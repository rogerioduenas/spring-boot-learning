# 🏋️ Exercise 13 — Resolving Bean Ambiguity with `@Primary`

## 📝 Description

A notification system can send messages via SMS or Email. Your task is to create two implementations of the same interface and use Spring to define one of them as the system's default implementation.

---

## ⚙️ Technical Objectives

- Reinforce the use of interfaces to achieve loose coupling.
- Master the `@Primary` annotation for resolving component ambiguity.

---

## 📂 Required Structure

### Interface

`NotificationService`

```java
void send(String msg);
```

### Implementations

- `SmsNotification`
- `EmailNotification`

Both classes must be annotated with `@Component`, and one of them must also be annotated with `@Primary`.

### Controller

- `NotificationController`

---

## 🎯 Endpoint

### POST `/notifications?msg=Hello`

---

## 📋 Functional Requirements

- `NotificationController` must inject only the `NotificationService` interface (without specifying any concrete implementation).
- When the endpoint is called, the `send()` method must be executed.
- The application must print the message to the console using the implementation marked with `@Primary`.
- Return **HTTP 200 OK** with the text:

```text
Notification processed
```

---

## 🧪 Postman Test Case

### Request

```http
POST http://localhost:8080/notifications?msg=Welcome
```

### Expected Result

- Status: **200 OK**
- The console should display the message generated by the implementation marked with `@Primary`.

Example:

```text
[SMS] Sending: Welcome
```
